Linked Lists


Motivation:
Limitations of fixed-size arrays or dynamically allocated arrays
Even dynamically allocated arrays are fixed sized
Resizeable arrays:
Automatically allocates new memory when array is full
Called vector in C++ stdlib
Some operations very quick, but insertion/deletion in the middle much slower
Access times very fast, adding/removing from end very fast

Goal: Represent collection of items for which inserting and removing in the middle is fast
Linked List:
Each node in linked list contains a value and a pointer to the next node
Doubly linked list contains pointer to both previous and next node
Traversal:
Continue until the pointer of a node is a null pointer
Linked list generally has pointer to head of list (to use it)
Should preserve head (or tail) pointer to avoid memory leak
Empty list represented by head pointer

Properties to prove when writing p->:
p cannot be uninitialzied
p cannot be nullptr
Check start and end of list behavior
Check one-item element behavior

Doubly Linked List:
Each node has a value, a next, and a previous pointer
