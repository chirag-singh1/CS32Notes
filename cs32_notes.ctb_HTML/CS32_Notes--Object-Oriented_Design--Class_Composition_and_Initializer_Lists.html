Class Composition and Initializer Lists


When a class contains member variables that are also classes
Composition Construction Rule:
C++ constructs member variables first (from top to bottom)
Then C++ constructs the overall/outer object
When an outer object contains member objects, C++ secretly adds code to the outer class's constructor to first call the default constructors of all member objects
Composition Destruction Rule:
C++ runs the outer class destructor first
C++ then runs the destructors of member variables in reverse order (bottom to top)
Implicitly adds destructor calls to member objects at end of outer destructor

Initializer List:
Uses colon to indicate initialization
Prevents implicit call to default constructor
Does NOT affect the order in which constructors are called
When multiple items in initializer list, must be placed in top to bottom order
Only needed when the body of the constructor is defined (not in the prototype for it)
Can be used for primitives as well
Parameters in initializer list do not have to be constant

Auto-Generated Constructors/Destructors:
Done to ensure that each member object is properly constructed and destructed
Ensures that by the time members are used in a class, they will be initialized
Ensures that when outer object goes away, member variables also go away
Auto-generated default constructor will not initialize primitive member variables
