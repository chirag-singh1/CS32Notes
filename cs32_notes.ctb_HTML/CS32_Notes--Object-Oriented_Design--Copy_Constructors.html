Copy Constructors


Solves problem of changes being made to a variable by pointer in local scope that persist to main thread
Example: array passed to function by pointer is then modified, and those issues will persist when they continue
Can create problem when first function calls delete on copy of function, but destructor is called again later (creates undefined behavior)
Occurs when multiple functions with different pointers share an object
Called automatically when an object is created from another object
When object is passed as a parameter by value
When object initialized to another object of same type
Returning value from a function (not pointer or reference)
