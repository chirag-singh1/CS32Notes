Destructors


Guarantees that allocated memory is freed when an object goes away
If memory is not freed, it will eventually cause a memory leak and a class
Destructors are unique to a class (cannot be overloaded)
Cannot take parameters
Implicit destructor will be defined by C++ if not implemented
Destructors are needed when a class allocates a system resource:
Reserves memory with new (deallocate memory with delete)
Opens a disk file (close file)
Connects to another computer over network (close connection)
Destructors called in specific cases:
Local objects destructors called when go out of scope
Local variables defined in block (like loop) destructed after block)
Dynamically allocated variables destructed when delete is called on them
Destructor called N times on an array of N objects
When the new keyword is not used, the destructor is called by default automatically
