Separate Compilation


Each source file (.cpp) produces an object function containing machine language translation for code and list of global names that require a definition somewhere else
Linker brings together all object files together into a single executable file
Rules:
Nothing can be defined more than once
Must use guards to prevent multiple compilation
Each need must be satisfied by some definition
Must be exactly one main routine

Happens when two classes depend on each other
Solved by using forward class declaration
Announces that a class is a name of a type
Incomplete type declaration defines type name, but no details about it
Requirement: When a type is declared, the compiler must know how much memory the type will include
This can be solved by using pointers because pointers all have the exact same size
Should use incomplete type specification whenever possible (speeds up compilation and simplifies dependencies)
Must include full header file in following cases:
Declare data member of that type (not a pointer)
Declare container of objects of class type
Create object of class type
Use member of that class type (call any method)
Does not require full header file:
Passing type as a parameter
Declare pointer to type
Declare array of pointers to that type
