Stacks and Queues


Higher-level data type
Can be implemented using linked list or array
Concept with restricted operations that are not trivial in array/list

Intuition: certain operations naturally are in a first in first out or first in last out order
Ex: LIFO order of operations for function calls (function stack)
Recursion

Stack: LIFO
Each new item taken out of the stack is the last one put in
Useful for iterative implementation of recursion
Could use array or linked list, but makes less readable
Declaring as a stack makes easier to understand
Operations:
Create empty stack
Add to empty stack (push)
Remove item from stack (pop)
Note that pop does NOT return a value
Fails if empty
Undefined behavior
Check top item of stack without removing (peek/top)
Fails if empty
Undefined behavior
Check if empty

Queue: FIFO
Each new item taken out of stack is the first one put in
Similar to line
Insertion at tail/back, remove at head/front
Often implemented with doubly linked list
Operations:
Create empty queue
Enqueue an item (add to tail)
Dequeue an item (remove from head)
Check top item at head of queue
Check if empty
